# FROG Bingo

你好，欢迎使用 FROG Bingo。本项目的设计主旨是把“每天最重要的三件大事”和生活 Bingo 小游戏结合起来，因此就像标题一样，主要分为两个部分：FROG，和 Bingo。

FROG 部分来自一个著名的时间管理理论：“吃掉那只青蛙”。在这个理论里，困难的任务就像吃青蛙，而你每天只需要吃掉三只最重要的青蛙，吃完你就可以 call it a day 了。在正式使用时，每天你都需要决定自己必须完成的 1-3 件任务。这些任务必须对你有意义，简单来说，做完这些任务之后，你得感觉今天活回本了。考虑到没人能不熬夜，在个人设置里，你可以自己决定这一天什么时候结束（但是不要超过次日六点）。

与此相关的还有一个问题：能不能不吃青蛙。青蛙，可爱又无害的小东西，不要吃它。于是本人将 FROG 作为首字母凑出四个单词，作为这个产品的概念：**F**orgive, **R**elax, **O**nward, **G**row. 

**F**orgive：放过你自己吧，你脑子有病。

**R**elax：做一点就可以躺了，加油。

**O**nward：总之，我们走在自己的方向上。虽然一直有猪打你，但这里是野炊，你走的方向都有意义。

**G**row：我也只是个几岁的成年人。She's young but she's daily growing. 

----------

Bingo 部分，那就是普通的 Bingo。然而其中还是有本人的一些小巧思：

1. 由于最初设计时只考虑了本人的情况，因此默认事件池的范围很狭窄，基本上只服务于ADHD-I型群体。好在现在的默认词条可以直接修改删除，用户可以自行修改或提高这些词条的难度。
2. 在完成一些 Bingo 后，所有人都可以获得自定义事件池，用户可以设定从不同事件池中抽取的词条数量，从而在不知道干啥的一天中找到一点方向（参考使用例：将一个自定义池设定为所有我没看完的书、影视作品、没打完的游戏）。
3. 随机生成的 Bingo 往往存在一些相对轻松的达成路线，为此引入了词条的权重（也即任务难度，最高为 3），以比较简陋的方式实现了各路线之间的相对平衡。任务难度也完全可定义和编辑，便于用户自行决定每个词条实现的难度。
4.  考虑到“运动30分钟”和“运动1小时”完全可以以不同权重的形式同时存在于事件池中，但不应在 Bingo 上同时出现，为此引入了互斥组功能，这样抽取的时候会避开同组的词条。
5. 引入热力图和连胜，使bingo具有更长效的使用价值和乐趣。
